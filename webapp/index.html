pingpong/40 [BROKEN]
Demonstrates how a request is ping-ponged between two handlers a specified number of times.
simple/sum/10/20
Adds two integers specified in the URL. The route sum/{a}/{b} will bind parameters a and b in the parameters map of the request map. In this example the handler receives the request and the parameters have to be picked out of the request map and cast to the correct type.
simple/sum/20/30
Adds two integers specified in the URL. The difference with the previous example is that the route sum-apply/{a}/{b}has an argument binding specification ('a|integer', 'b|integer') that is used to prepare an argument list for the handler function. This also takes care of casting the string values from the request map to correct types. In this way a normal function can be used as a request handler.
